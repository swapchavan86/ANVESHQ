# Nexara: Next-Era Market Intelligence Platform

Nexara is a full-stack market intelligence platform designed to provide daily insights into the stock market. It features a Python backend for data analysis and a React-based frontend for user interaction.

## Project Structure

The project is organized into four main parts: a `Backend`, a `UI`, a `Test` suite, and a `data` directory.

```
d:\Projects\Nexera\
├───Backend\
│   ├───src\
│   └───...
├───UI\
│   ├───src\
│   └───...
├───Test\
│   └───...
├───data\
│   └───...
├───.gitignore
├───README.md
└───...
```

- **`Backend/`**: Contains the Python-based backend responsible for data fetching, analysis, and providing an API.
  - **`src/`**: The main source code for the backend.
- **`UI/`**: The React-based frontend application for displaying data and user interaction.
  - **`src/`**: The main source code for the frontend.
- **`Test/`**: Contains the test suite for the project, primarily for the backend. It uses `pytest` for testing.
- **`data/`**: This directory is used for storing data artifacts, such as the master stock list generated by the backend.

## Features

- **Weekly Stock Universe Builder**: A recurring job that compiles a canonical list of stocks.
- **Daily Intelligence Engine**: Scans the market daily to identify trading signals based on various configurable parameters.
- **RESTful API**: The backend exposes endpoints for the frontend to consume market data.
- **React-based UI**: A modern and responsive user interface for visualizing market intelligence.
- **User Management**: The application includes features for user sign-up, sign-in, and management.

## Setup and Installation

### Backend (Python)

The backend uses `pydantic-settings` for configuration management. Settings can be provided via environment variables or a local `.env` file.

1.  **Navigate to the Backend directory:**
    ```bash
    cd d:\Projects\Nexera\Backend
    ```
2.  **Create a virtual environment:**
    ```bash
    python -m venv venv
    ```
3.  **Activate the virtual environment:**
    - On Windows:
      ```bash
      venv\Scripts\activate
      ```
    - On macOS/Linux:
      ```bash
      source venv/bin/activate
      ```
4.  **Install dependencies:**
    ```bash
    pip install -r requirements.txt
    ```
5.  **Configure environment variables:**
    Create a `.env` file in the `Backend` directory. You can copy the example if one is provided. The following variables are required:

    - `MODE`: `DEV`, `PROD`, or `TEST` (Default: `DEV`)
    - `DATABASE_URL`: Connection string for the primary database.
    - `TEST_DATABASE_URL`: Connection string for the test database.
    - `LOG_LEVEL`: `DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL` (Default: `INFO`)
    - And other configuration variables as listed in `Backend/src/config.py`.

### Frontend (React)

The frontend is a React application built with Vite.

1.  **Navigate to the UI directory:**
    ```bash
    cd d:\Projects\Nexera\UI
    ```
2.  **Install dependencies:
    ```bash
    npm install
    ```
3.  **Start the development server:**
    ```bash
    npm run dev
    ```

**Troubleshooting PostCSS Errors:**

If you encounter PostCSS errors, follow these steps for a clean re-installation:

1.  Stop the development server (`Ctrl+C`).
2.  Delete `node_modules`, `package-lock.json`, and `.vite` cache.
    ```bash
    rmdir /s /q node_modules
    del package-lock.json
    rmdir /s /q .vite
    ```
3.  Reinstall dependencies and start the server:
    ```bash
    npm install
    npm run dev
    ```

## Running Tests

The test suite for this project is located in the `Test/` directory and uses the `pytest` framework.

### Backend

To run the backend tests, execute the following command from the root project directory:

```bash
pytest
```

This will automatically discover and run the tests in the `Test/` directory. Ensure your `TEST_DATABASE_URL` is correctly configured in your `.env` file or environment variables, as the tests may require a separate database.

## Usage

- **Backend**: To run the backend application, execute the main script:
  ```bash
  python Backend/src/main.py
  ```
- **Frontend**: Access the application through your web browser at the address provided by the Vite development server (usually `http://localhost:5173`).
